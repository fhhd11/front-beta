# etrnl.chat - AI Chat Platform

## Суть проекта

**etrnl.chat** — это платформа для создания персонализированных AI-ассистентов с долгосрочной памятью и способностью к обучению. В отличие от обычных чатботов, которые "забывают" пользователя после каждой сессии, наши агенты развиваются вместе с пользователем, запоминая предпочтения, стиль общения и важные детали.

### Ключевая проблема
Современные AI-ассистенты страдают от:
- **Ограниченного контекста** — забывают предыдущие разговоры
- **Отсутствия персонализации** — одинаковый опыт для всех пользователей  
- **Фрагментированной памяти** — не могут связывать информацию между сессиями
- **Отсутствия эволюции** — не становятся лучше со временем

### Наше решение
Создание **stateful AI-агентов** на базе Letta (MemGPT), которые:
- ✅ **Помнят все** — бесконечная история диалогов с интеллектуальным управлением памятью
- ✅ **Адаптируются** — изучают стиль пользователя и подстраиваются под него
- ✅ **Эволюционируют** — становятся умнее и полезнее с каждым взаимодействием  
- ✅ **Персонализированы** — уникальный агент для каждого пользователя

## Продукт

### MVP: Персональный Ассистент
**Один умный агент на пользователя** с возможностями:
- Бесконечный диалог с полной историей
- Автоматическое обучение предпочтениям пользователя
- Интеллектуальное управление памятью (важное остается, лишнее архивируется)
- Sleep-time обработка — агент "думает" и улучшает память даже когда вы не общаетесь

### V2: Мульти-контекстная Система  
**Расширение до множественных агентов:**
- **Основной чат** — повседневное общение
- **Проектные агенты** — специализированные помощники для работы, обучения, творчества
- **Кросс-проектные связи** — агенты обмениваются insights между контекстами
- **Умное переключение** — автоматические предложения перехода к релевантному агенту

### V3: AI Workspace
**Полноценная рабочая среда:**
- Интеграция с внешними инструментами и API
- Командная работа с общими агентами
- Advanced analytics использования и эффективности
- Marketplace расширений и специализированных агентов

## Техническая архитектура

### Современный микросервисный подход:
- **Zuplo** — API Gateway с проксированием к LiteLLM
- **Agent Management Service** — управление жизненным циклом агентов (наш репозиторий)
- **Supabase** — PostgreSQL + auth + Edge Functions
- **Letta Self-hosted** — stateful agent engine с MemGPT архитектурой
- **LiteLLM** — унифицированный доступ к различным LLM провайдерам

### Ключевые преимущества архитектуры:
- **Версионирование агентов** — обновления без потери пользовательских данных
- **Horizontal scaling** — каждый компонент масштабируется независимо  
- **Provider flexibility** — легкое переключение между LLM провайдерами
- **Data ownership** — полный контроль над пользовательскими данными
